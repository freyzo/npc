Why

Stop context switching

The problem

You're deep in code and need to message someone on Slack, check Discord, or reply to an email. Every time, you leave your editor, lose focus, and break flow.


Browser automation tools want API keys, OAuth tokens, or separate bot accounts for every service. That's a lot of setup for "send a message."

NPC fixes this

One CLI + one Chrome extension. Your IDE agent says what to do ("message Anna on Slack"), NPC does it in your real, logged-in browser.


No API keys per service. No OAuth. Uses the browser sessions you already have. Works with any MCP IDE - Cursor, VS Code, Windsurf.

How it works

Three pieces, one pipeline

Your IDE handles reasoning. NPC is the hands - it executes browser actions via Chrome DevTools Protocol. No LLM inside NPC.

Get started

Three steps, two minutes

1

Install the CLI

npm i -g npc-agent

2

Load the extension

Open chrome://extensions, enable Developer mode, click Load unpacked, select the extension/ folder.

3

Add to your IDE

Add NPC to .cursor/mcp.json or .vscode/mcp.json and start asking your agent to control the browser.
